# Using Easywork

**在选择技能之前，必须先阅读此技能。**

帮助智能体根据任务类型选择正确的技能，避免内联处理本应由专门技能完成的工作。

此技能是路由器，不替代下游技能。它的职责是决定使用哪个技能，然后立即调用。

## 核心规则

**先识别任务类型，再选择技能。**

- 如果任务是对仓库进行系统性 bug 巡查或代码审查，选择 `bug-sweep`。
- 如果任务不属于任何已有技能，不要使用此技能。

## 硬性规则

- 如果任务可能属于本仓库的某个技能，先用此技能路由，再继续。
- 不要因为查询看起来简单就跳过路由，直接内联处理。
- 一旦确定了正确的下游技能，立即调用它。

## 路由表

| 任务类型 | 技能 |
|---------|------|
| 对仓库执行系统性 bug 巡查、代码审查或质量审计 | `bug-sweep` |

## 路由流程

1. 判断任务是否属于本仓库的某个技能
2. 如果是，在路由表中查找对应技能
3. 立即调用选中的技能
4. 如果不是，不使用此技能

## 红旗

以下想法通常意味着智能体即将绕过路由器：

| 想法 | 应对 |
|------|------|
| "只是查一下 bug，我直接看就行。" | 先路由。bug 巡查有专门的工作流，应通过 `bug-sweep` 执行。 |
| "我已经知道该用哪个技能，可以即兴发挥。" | 严格遵循路由规则，然后调用选中的技能。 |

## 非目标

此技能不是：

- 每次对话都必走的通用启动技能
- 下游技能工作流的替代品
- 通用规划或实现技能

此技能只负责选择和调用，下游技能拥有实际工作流。

## 交接规则

决定正确路径后：

1. 立即调用选中的技能
2. 不要在交接过程中将 `Inference`、`Open Question` 或 `Decision Blocker` 简化为已确认事实
